## Key Learnings

### macOS Code Signing (Critical)

Apple Silicon enforces code signature validation. Modified binary = invalid hash = SIGKILL.

**Problem**: `codesign --force --sign -` (ad-hoc) produces a smaller `__LINKEDIT` section than the original Developer ID signature. File size changes → Bun can't locate JS bundle → falls back to evaluating CLI args as JavaScript.

**Solution**: Use `--signature-size` flag to force codesign to allocate enough space, matching the original file size exactly. The correct value is found by binary search (~1162185 for this version, but varies slightly per run).

**How Bun locates JS**: Bun uses a trailer marker `---- Bun! ----` near the end of the `__BUN` section, with metadata offsets relative to file structure. The Mach-O section headers (`__BUN` offset/size) are NOT modified by codesign, but EOF-relative calculations break if file size changes.

### Size Constraints

Patches must not increase JS bundle size (Mach-O section sizes are fixed in headers). Strategy: trim existing content to make room for new content. Equal-length replacement is safest.

### Anchor Stability

String constants in the compact prompt are stable across minification — they're literal strings, not variable names. Good anchor candidates:
- `"Your task is to create a detailed summary of the conversation so far"`
- `"There may be additional summarization instructions"`  
- `"1. Primary Request and Intent"`
- Section numbering (1-9) in the compact prompt template
